## 1. 里程碑总览

## M1：开档与世界生成闭环（P0）

- 目标：玩家可创建存档并进入首回合。
- 输出：世界卡可选、模型可测、世界可生成、存档可落盘。

## M2：地图探索与 3+1 对话闭环（P0）

- 目标：玩家能移动、对话并推进回合。
- 输出：地图可走、回合可推进、三选项+自定义输入可用。

## M3：事件引擎 v1 与回放分叉（P0）

- 目标：事件系统驱动状态变化，存档可回放与分叉。
- 输出：触发规则/动作执行、日志可回放、历史可另存新档。

## M4：主题、导入导出、稳定性（P1）

- 目标：完成体验层与非功能性增强。
- 输出：三主题、世界卡导入导出、错误处理完善。

## 2. 工作分解结构（WBS）

## 2.1 前端（Vue3 + shadcn/vue）

1. `FE-001` 项目骨架与路由
- 内容：主菜单、游戏页、世界卡管理、设置页路由。
- 依赖：无。
- 验收：页面可跳转，基础布局稳定。

2. `FE-002` 主菜单与存档列表 UI
- 内容：新建、继续、删除存档界面。
- 依赖：`BE-003 list_saves`、`BE-004 delete_save`。
- 验收：可创建并显示存档元信息。

3. `FE-003` 新建存档向导
- 内容：世界卡选择、角色输入、模型参数设置、创建动作。
- 依赖：`BE-001 create_save`、`BE-006 generate_world`。
- 验收：创建成功后进入游戏界面。

4. `FE-004` 游戏主界面布局
- 内容：地图区、叙事区、选项区、状态栏。
- 依赖：`FE-001`。
- 验收：四区联动且状态更新正确。

5. `FE-005` 地图 Canvas 组件
- 内容：节点渲染、边渲染、可达高亮、点击移动。
- 依赖：`BE-008 move_to_location`。
- 验收：只能移动到邻接且可达节点。

6. `FE-006` 对话 3+1 交互组件
- 内容：三选项按钮、自定义输入框、提交后加载状态。
- 依赖：`BE-007 run_turn`。
- 验收：任意一种输入都能推进回合。

7. `FE-007` 事件反馈与状态变化 UI
- 内容：toast、日志条目、关系变化提示。
- 依赖：`ENG-004 事件回包结构`。
- 验收：每次触发事件可见反馈。

8. `FE-008` 世界卡管理页
- 内容：列表、创建、复制、导入导出入口。
- 依赖：`BE-009 import_world_card`、`BE-010 export_world_card`。
- 验收：卡片导入后可用于新建存档。

9. `FE-009` 模型设置页
- 内容：Provider 选择、模型参数、连通性测试。
- 依赖：`BE-011 test_model_provider`。
- 验收：能区分鉴权错误与网络错误。

10. `FE-010` 主题系统与 token
- 内容：三套主题变量、切换持久化。
- 依赖：`FE-001`。
- 验收：主题切换全局生效且重启保留。

## 2.2 后端（Rust + Tauri Commands）

1. `BE-001` `create_save(config) -> SaveMeta`
- 内容：创建存档目录、写入 meta 与初始 snapshot。
- 依赖：`ENG-001 存储结构`。
- 验收：重复 ID 不冲突，创建幂等可控。

2. `BE-002` `load_save(save_id) -> SaveBundle`
- 内容：读取并返回快照与必要日志片段。
- 依赖：`ENG-001`。
- 验收：损坏存档能返回明确错误。

3. `BE-003` `list_saves() -> Vec<SaveMeta>`
- 内容：遍历存档目录并按更新时间排序。
- 依赖：`BE-001`。
- 验收：列表字段完整且排序正确。

4. `BE-004` `delete_save(save_id) -> ()`
- 内容：删除指定存档。
- 依赖：`BE-001`。
- 验收：不存在存档返回可识别错误码。

5. `BE-005` Provider 适配层
- 内容：统一 OpenAI/Claude 请求模型与错误映射。
- 依赖：无。
- 验收：两 Provider 返回结构统一。

6. `BE-006` `generate_world(input) -> WorldInit`
- 内容：拼接系统提示词层级并生成初始世界。
- 依赖：`BE-005`、`ENG-003 Prompt 规范`。
- 验收：输出满足 schema 且包含初始地点/NPC/剧情钩子。

7. `BE-007` `run_turn(turn_input) -> TurnResult`
- 内容：执行单回合，调用模型、判定事件、落盘日志。
- 依赖：`ENG-004 事件引擎`、`BE-005`。
- 验收：同回合重生成不重复持久化。

8. `BE-008` `move_to_location(save_id, location_id) -> MoveResult`
- 内容：验证路径可达性，更新玩家位置并触发进入事件。
- 依赖：`ENG-004`。
- 验收：跨越不可达节点必须拒绝。

9. `BE-009` `import_world_card(file) -> WorldCard`
- 内容：解析 JSON、校验 schema、保存卡片。
- 依赖：`ENG-002 Schema`。
- 验收：版本不兼容时提示升级策略。

10. `BE-010` `export_world_card(card_id, file) -> ()`
- 内容：导出卡片 JSON（保留 schemaVersion）。
- 依赖：`BE-009`。
- 验收：导出文件可再次导入成功。

11. `BE-011` `test_model_provider(config) -> ConnectivityResult`
- 内容：最小请求探测，返回连通性与耗时。
- 依赖：`BE-005`。
- 验收：错误类型可区分：鉴权/网络/限流/超时。

## 2.3 引擎与共享模块（Domain）

1. `ENG-001` 存储层与文件布局
- 内容：目录组织、读写锁策略、原子写入。
- 验收：异常中断不产生半写入文件。

2. `ENG-002` 数据模型与 schema 校验
- 内容：WorldCard、SaveSnapshot、EventLogEntry、GameEvent schema。
- 验收：非法字段可定位到路径级错误。

3. `ENG-003` Prompt 拼装器
- 内容：系统层/世界层/存档层/回合层合成，附 token 预算器。
- 验收：超过预算自动压缩中期记忆。

4. `ENG-004` 事件引擎 v1
- 内容：触发器扫描、守卫判断、动作执行、链式触发、防循环。
- 验收：链深限制和去重键生效。

5. `ENG-005` 记忆系统
- 内容：短期窗口、中期摘要、长期事实锁。
- 验收：关键事实在 100 回合后仍一致。

6. `ENG-006` 回放与分叉
- 内容：基于 `events.ndjson` 重建状态与另存分支。
- 验收：分支回放结果与源回合前缀一致。

## 5. 完成定义（按模块）

1. 后端模块 DoD
- 有命令级测试用例。
- 错误码与错误文案标准化。
- 文件读写具备原子性和回滚保护。

2. 前端模块 DoD
- 所有主流程具备加载/错误/空状态。
- 无阻塞操作都支持用户取消。
- 核心交互具备键盘可达性。

3. 引擎模块 DoD
- 事件链可追踪、可复现、可解释。
- 记忆压缩不破坏长期事实。
- 回放和分叉结果一致。

## 6. 风险清单与缓解动作

1. 模型响应结构不稳定
- 缓解：强制 JSON schema 输出 + 重试清洗。

2. 事件规则复杂度快速膨胀
- 缓解：限制动作原语；事件评审模板；规则 lint。

3. 日志量增长导致 I/O 压力
- 缓解：ndjson 分片、按回合归档、按需加载。

4. 多 Provider 行为差异
- 缓解：构建回归样例集，固定 20 个基准场景对齐。
